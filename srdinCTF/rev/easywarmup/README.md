# easywarmup - writeup

## Flag

`srdnlen{pl5_Charles_w1n_th3_champ1on5hip}`

## Reversing notes

The binary is a stripped Windows x64 PE (`rev_artistic_warmup.exe`).

### 1) Find the check site

`strings` reveals only:
- `Invalid flag.`
- `Valid flag!`

Disassembly around the xref to `Invalid flag.` shows the core compare loop:

- It compares `0x15f90` bytes (`90000`) from a generated buffer.
- Check logic per byte:
  - `if ((generated[i] ^ 0xAA) != blob[i]) fail`
- So expected generated bytes are: `blob[i] ^ 0xAA`.

### 2) Resolve hashed WinAPI calls

A custom resolver at `0x140001510` hashes export names with seed `0x1505`:

`h = h + (h << 5) + c`

Resolved API hashes in the verifier map to:
- `CreateCompatibleDC`
- `CreateDIBSection`
- `CreateFontA`
- `SelectObject`
- `SetBkColor`
- `SetTextColor`
- `TextOutA`
- `GdiFlush`
- `DeleteObject`
- `DeleteDC`

So the input is rendered to a bitmap and compared against an embedded reference.

### 3) Recover bitmap parameters

From stack setup before `CreateDIBSection`:
- `BITMAPINFOHEADER.biSize = 40`
- `biWidth = 450`
- `biHeight = -50` (top-down)
- `biPlanes = 1`
- `biBitCount = 32`

Thus compared image is a `450x50` BGRA buffer (`450 * 50 * 4 = 90000`), matching `0x15f90`.

### 4) Extract reference image

```python
from pathlib import Path
from PIL import Image

p = Path('rev_artistic_warmup.exe').read_bytes()
blob = p[0xC5020:0xC5020 + 0x15F90]
expected = bytes(b ^ 0xAA for b in blob)

img = Image.frombytes('RGBA', (450, 50), expected, 'raw', 'BGRA')
img.save('target.png')
```

Viewing the decoded rendered text reveals the flag.

## Why this works

The checker does not parse a textual flag directly; it validates the **rendered pixels** of your input text. Reconstructing the target bitmap and reading the rendered phrase gives the required flag.
